SoTデータのSoT循環

Notionが唯一のSoT。調理場(Orb)・器具(Skills)・見学窓(MC)は様式を変えて取り出すだけのミラー層。1情報=1SoT原則で、データの実体は常に倉庫に1つ。

SESセッション周回

rehem→作業→hem→handoffの周回。調理場で作り→配管(Git)で運搬→倉庫のセッションログDBに採番・記録。RPG精算とhandoff Index DBで履歴が腐葉土として残る。

PSH通知パイプ

倉庫の inbox 件数を Discordへ毎朝7時 push(task-inbox-daily cron, 稼働中)。カレンダー連携は整備途中=点線。最終判断は人間レーンへ。

ASSET成果の堆積

調理場・配管を経た成果が GitHub Pages / Zenodo / KDP に公開・堆積。消化型でなく装飾型で残し、資産は薄い循環で倉庫(SoT)へメタ情報を還す。
